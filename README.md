DISA2 — это веб-приложение для учёта заявок, встреч и приёмки автомобилей в сервисе. Проект включает авторизацию, роли `user` / `admin`, админ-панель, просмотр логов активности, управление пользователями и адаптацию под телефон и планшет.

## 🚀 Что умеет сайт

### Для обычного пользователя
- регистрация и вход в систему;
- создание новой заявки / встречи;
- просмотр своих и общих заявок;
- открытие страницы приёмки автомобиля;
- работа с отчётом и карточкой осмотра.

### Для администратора
- просмотр **всех** заявок;
- изменение статусов (`Новая`, `В работе`, `Завершена`, `Отменена`);
- открытие отчётов и контроль заявок;
- просмотр журнала действий;
- полноэкранное управление пользователями:
  - смена логина;
  - смена email;
  - смена роли;
  - установка нового пароля;
  - удаление пользователя.

> Пароли в базе **не хранятся в открытом виде** — используется `bcrypt`-хэширование.

## 🗄️ Настройка базы данных PostgreSQL

### 1. Создайте базу данных
Например, с именем `auth_db`:

```sql
CREATE DATABASE auth_db;
```

### 2. Примените структуру из файла `database/schema.sql`

Если используете `psql`:

```bash
psql -U postgres -d auth_db -f database/schema.sql
```

Если используете **pgAdmin**:
1. Откройте нужную базу.
2. Откройте `database/schema.sql`.

Скрипт создаёт таблицы:
- `users`
- `meetings`

## 🔐 Настройка `.env`

В корне проекта создайте файл `.env`.

Пример содержимого:

```env
PORT=3000
HOST=0.0.0.0

DB_HOST=localhost
DB_PORT=5432
DB_NAME=auth_db
DB_USER=postgres
DB_PASSWORD=your_postgres_password

JWT_SECRET=change_this_to_a_long_random_secret
```

### Что означают переменные

| Переменная | Значение |
|---|---|
| `PORT` | порт запуска сервера, обычно `3000` |
| `HOST` | `0.0.0.0`, если нужен доступ по локальной сети |
| `DB_HOST` | адрес PostgreSQL, обычно `localhost` |
| `DB_PORT` | порт PostgreSQL, обычно `5432` |
| `DB_NAME` | имя базы, например `auth_db` |
| `DB_USER` | пользователь PostgreSQL |
| `DB_PASSWORD` | пароль пользователя PostgreSQL |
| `JWT_SECRET` | секрет для токенов авторизации |

> Если `.env` не задан, приложение попробует использовать локальные значения по умолчанию из `src/config/db.js`, но лучше всегда указывать свои реальные данные явно.

## 👤 Как войти в систему и получить админ-доступ

### Регистрация обычного пользователя
1. Откройте `http://localhost:3000/register.html`
2. Создайте аккаунт.
3. После этого можно войти через `index.html`.

### Важно про администратора
В репозитории **нет жёстко зашитого логина и пароля администратора**.

Это значит:
- по умолчанию все новые аккаунты создаются с ролью `user`;
- **стандартного admin/admin нет**;
- пароль администратора — это пароль, который вы сами задали при регистрации этого аккаунта.

### Как сделать пользователя админом
После регистрации выполните SQL в PostgreSQL:

```sql
UPDATE users
SET role = 'admin'
WHERE username = 'ВАШ_ЛОГИН';
```

После этого:
1. войдите под этим аккаунтом;
2. система перенаправит вас в `admin.html`.

### Если забыли пароль администратора
Открытый пароль из базы получить нельзя, потому что хранится только `bcrypt`-хэш.  
Варианты:
- задать новый пароль через админ-панель, если вход уже доступен;
- создать нового пользователя и выдать ему роль `admin` через SQL.

## 🧭 Как пользоваться сайтом

### 1. `index.html` — вход
Здесь пользователь вводит логин и пароль.  
После успешного входа:
- `admin` → попадает в `admin.html`
- `user` → попадает в `user.html`

### 2. `register.html` — регистрация
Создание нового аккаунта.

Требования:
- логин минимум **3 символа**;
- корректный email;
- пароль минимум **6 символов**.

### 3. `user.html` — кабинет пользователя
Можно:
- создать заявку;
- смотреть список встреч;
- открывать карточку приёмки;
- работать со своими данными в рамках доступных прав.

### 4. `admin.html` — админ-панель
Можно:
- видеть все встречи;
- менять статусы;
- смотреть отчёты;
- открывать логи;
- через кнопку `👥` управлять пользователями в полноэкранном режиме.

### 5. `reception.html` — приёмка
Страница приёмки автомобиля, осмотра и работы с деталями заявки.

## 🔄 Что и как здесь "манипулируется"

Ниже кратко описано, что делает система внутри.

### Авторизация
- маршрут `POST /api/auth/register` — регистрация;
- маршрут `POST /api/auth/login` — вход;
- маршрут `POST /api/auth/logout` — выход;
- JWT-токен хранится в `localStorage` и отправляется в `Authorization: Bearer ...`.

### Пользователи
- все пользователи лежат в таблице `users`;
- роли: `user` и `admin`;
- админ может менять:
  - `username`
  - `email`
  - `role`
  - пароль
  - удалять аккаунты

### Заявки / встречи
- лежат в таблице `meetings`;
- создаются через `POST /api/meetings`;
- список загружается через `GET /api/meetings`;
- статус меняется через `PUT /api/meetings/:id/status`.

### Логи и онлайн-активность
- работают через `public/monitor.js`;
- админ видит логи действий;
- индикатор онлайна теперь показывается **только администратору**.

## ❗ Возможные проблемы и решения

### Сервер не стартует
Проверьте:
- установлен ли Node.js;
- выполнен ли `npm install`;
- не занят ли порт `3000`.

### Ошибка подключения к PostgreSQL
Проверьте:
- запущен ли PostgreSQL;
- правильные ли `DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USER`, `DB_PASSWORD` в `.env`;
- создана ли база `auth_db`;
- выполнен ли `database/schema.sql`.

### Ошибка `JWT_SECRET не настроен`
Добавьте в `.env` строку:

```env
JWT_SECRET=your_secret_key
```

### Телефон не открывает сайт по Wi-Fi
Проверьте:
- что сервер запущен с `HOST=0.0.0.0`;
- что вы используете IP-адрес из консоли сервера;
- что Windows Firewall не блокирует входящие подключения на порт `3000`.
